# CLAUDE.md

## Project Overview

Scoundrel is a solitaire roguelike card game built in C++17 with SDL2. It compiles to native desktop, WebAssembly (via Emscripten), and Miyoo A30 handheld. The web version is deployed to GitHub Pages.

## Build Commands

All builds run from `src/`:

```bash
cd src
make              # native build -> SCOUNDREL binary
make run          # build and run native
make js           # Emscripten/WASM build -> outputs to ../web/
make miyooa30     # Miyoo A30 build (inside Docker container only)
make clean        # remove all build artifacts
make clean_nonlib # remove non-library build artifacts only
```

CI runs `scripts/ci-build.sh` which does `cd src && make js`.

```
  game/

    Dispatch.cpp/h      # Action queue (singleton); actions added then executed each frame
    state.h             # All game state structs, enums, layout constants
    calculations.hpp    # Combat/heal math
    actions/

    sdl2wrapper/        # SDL2 abstraction layer
      Window.cpp/h      # SDL window, renderer, event loop, drawing primitives
      Events.cpp/h      # Event routing (keyboard, mouse callbacks via EventRoute stack)
      Store.cpp/h       # Asset storage (sprites, textures, sounds, fonts)
      Sprite.cpp/h      # Sprite data
      Animation.cpp/h   # Sprite animation
      Localization.cpp/h # L10n via JSON files

  scaffold.js/css       # On-screen D-pad/buttons, canvas layout, orientation handling

```

## Architecture

### Input Flow

All input resolves to action dispatches through `GameManager`:

- **Keyboard**: SDL_KEYDOWN -> `Events::keydown` -> EventRoute `onkeypress` -> `GameManager::handleKeyPress`
- **Mouse/Touch**: SDL_MOUSEBUTTONDOWN -> `Events::mousedown` -> EventRoute `onmousedown` -> `GameManager::handleMouseClick`
- **Web scaffold buttons**: JS `Lib.handleButtonDown` -> `Module.ccall('setKeyDown')` -> `Events::keydown` (same as keyboard)
- **Gamepad**: JS polling in `lib.js` -> `Lib.handleButtonDown` (same as scaffold)

### Game State Machine

`InputMode` enum in `state.h` controls which inputs are accepted:

- `MENU` -> any input starts game
- `ROOM` -> navigate/select room cards or flee button
- `CONFIRM` -> Yes/No or Weapon/Fist selection
- `NONE` -> animations playing, input disabled
- `END_GAME_SCREEN` -> any input returns to menu

### Action System

Actions inherit from `AbstractAction` and are queued via `Dispatch::get().addAction(...)`. The macro `DISPATCH_ACTION(ActionType, args...)` is shorthand. All queued actions execute in `Dispatch::update()` each frame. Async actions with timers handle animations.

### Rendering

The game renders at 640x480 into an intermediate SDL render target. Cards are 32x48 pixels at 3x scale (96x144 rendered). All positions are defined as constants in `state.h`. `Render.cpp` draws cards, UI buttons, cursor, health, and screen overlays. Sprites use centered drawing by default.

### Key Constants (state.h)

- `CARD_WIDTH=32`, `CARD_HEIGHT=48`, `CARD_SCALE=3`
- `BUTTON_WIDTH=64`, `BUTTON_HEIGHT=12` (ui_button sprite)
- `WINDOW_WIDTH=640`, `WINDOW_HEIGHT=480`
- Named position constants: `FLEE_UI_POS`, `CONFIRM_LEFT_POS`, `WEAPON_CARD_POS`, etc.

## Platform-Specific Code

- `#ifdef __EMSCRIPTEN__` — Web/WASM: emscripten main loop, KEEPALIVE exports (setKeyDown, setKeyUp, etc.)
- `#ifdef __vita__` — Vita/joystick input handling (also used conceptually for Miyoo A30 gamepad)
- `#ifdef MIYOOA30` — Miyoo A30: rotated render target, alternate SDL include paths

## Code Style

- LLVM-based `.clang-format`: 2-space indent, 80-column limit, pointers left-aligned
- C++17 standard
- Namespaces: `program` (game code), `sdl2w` (SDL2 wrapper), `program::actions` (action classes)
- Header-only action classes in `.hpp` files
- No CMake; single Makefile

## Localization

Strings are wrapped in `LOCSTR("...")` macro and translated via `sdl2w::L10n::trans()`. Generate localization files with:

```bash
python3 scripts/scan_locstr.py --languages la en
```

## CI/CD

GitHub Actions (`.github/workflows/pages.yaml`): on push to `main`, builds WASM with Emscripten 2.0.10 and deploys `web/` directory to GitHub Pages.
